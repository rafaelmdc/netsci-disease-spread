Data & networks

Everything the simulator needs, built from scratch without the command line. Work top to bottom — each step feeds the next. Actions run as background jobs; the status below updates as they finish.

Pipeline
{# ---- step 1: retrieve raw sources ---------------------------------- #}
1
Retrieve sources

Downloads the raw airport / city / ferry datasets the networks are built from, into data/raw/. Needs internet and can take a few minutes — this is the slowest first step.

{# ---- step 2: build networks ---------------------------------------- #}
2
Build networks · needs step 1

Turns the raw sources into the processed graphs under data/processed/ that the simulator runs on. Nothing can simulate until at least one network exists. "Build all" is heavy; use "Build one" to make a single region/layer combination quickly.

{# ---- step 3: run simulations (lives on other pages) ---------------- #}
3
Run simulations · needs a network from step 2
New run Study Run sweep

Each simulation writes a run under results/. New run launches one; Study runs a small parameter study; Run sweep (on the Compare → Aggregate tab) runs the full experiment.yaml grid — the canonical but heavy option.

{# ---- step 4: aggregate ---------------------------------------------- #}
4
Aggregate tables · needs runs from step 3

Rolls every run's summary into summary.parquet and each network's structure into structure.parquet — the tables behind the charts on the Compare → Aggregate tab. Cheap; re-run it after adding runs.
