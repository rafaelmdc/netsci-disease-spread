\section{Methodology}
Our question is how different disease types spread across the transport network that connects
Europe, and which network-science protection best contains them. Answering it fairly requires
three components built in sequence: a transport network from open data (Section~\ref{sec:map}), a
metapopulation epidemic engine that runs on it (Section~\ref{sec:outbreak}), and the disease types
and protection strategies evaluated on top (Sections~\ref{sec:spread} and~\ref{sec:defend}). One
principle constrains all of them: in any comparison, only the factor under study varies while
everything else is held fixed. The study is therefore comparative rather than predictive: we do not
estimate how many people a real epidemic would infect, but quantify differences between scenarios
under identical conditions. The whole pipeline, from data retrieval through network generation to
evaluation, is a single reproducible workflow (Figure~\ref{fig:pipeline}).

\subsection{Building the map} \label{sec:map}
The substrate is a weighted graph $G=(V,E)$ in which nodes $v\in V$ are cities and edges
$(i,j)\in E$ are travel routes carrying weight $w_{ij}$ proportional to traffic. We construct $G$
with NetworkX~\cite{hagberg:networkx} from four open sources: air routes from
OpenFlights~\cite{openflights}, city populations from GeoNames, road and rail topology from
OpenStreetMap and GRIP~\cite{meijer:grip}, and ferry routes from OpenStreetMap. Region is a
parameter of construction, which lets us hold the construction \textbf{method} fixed while varying
only geography.

Because a single air layer misrepresents how an epidemic actually propagates, the same node set
carries up to three layers, air, land, and water, combined into one shared-node multilayer network
(Figure~\ref{fig:stack}). Edge weights are layer-specific passenger-volume proxies, but they come
from different recipes according to what data exists. Air and water weights are real route
frequencies: OpenFlights for air, and an OpenStreetMap ferry dump for water, filtered to
passenger-carrying services (OSM tag \texttt{passenger} not equal to \texttt{no}) over crossings at
least 5~km long, which drops freight-only and short canal hops while keeping genuine straits and sea
ferries. Land, which has no comparably uniform cross-region flow data, is instead modelled with a
radiation kernel $K_{ij} = \mathrm{flux}_{ij}/\mathrm{pop}_i$~\cite{simini:radiation} computed from
populations. Fairness here requires not that all layers share one recipe, but that each layer is
built by a single method applied identically to every region, so that differences between regions
reflect topology rather than the data source; European commuting data (Eurostat) is reserved for
validating the land kernel, not for parameterising any region. Node populations $N_i$ come from
GeoNames where available, with a degree-based proxy $N_i = P_0 + d(i)\,P_\mathrm{route}$ as a
fallback.

Before running any outbreak we characterise each built network structurally, since topology bounds
what an epidemic or an intervention can do. For every network we measure order and density, the
degree distribution and its heterogeneity ratio $\langle k^2\rangle/\langle k\rangle$ (which
governs the metapopulation invasion threshold~\cite{colizza:invasion}), degree assortativity,
average clustering, the mean shortest-path length and diameter, community modularity, and the rank
correlation $\rho$ between a city's degree and its betweenness, together with the count of
\emph{anomalous gateways}, low-degree cities whose betweenness far exceeds that of their
similar-degree peers. These gateways are the anomalous centrality that Guimer\`a et al.\ found in
the worldwide air network~\cite{guimera:anomalous}; we flag them not by an arbitrary cutoff but by
standardising each city's betweenness against that of its similar-degree peers and testing for a
significant excess under Benjamini--Hochberg false-discovery-rate control, the degree-conditioned
benchmark of~\cite{sun:anomalous}. These descriptors are reported per network and read directly
against the epidemic results.

\subsection{Running an outbreak on the map} \label{sec:outbreak}
Each city runs a compartmental model internally, while individuals move between cities along
the edges of $G$ once per day. This is a metapopulation reaction--diffusion process~\cite{colizza:reactiondiffusion}:
a local \textbf{reaction} updates the compartments within each city, and a
\textbf{mobility} step couples cities through the network. Within city $i$ the force of
infection is $\lambda_i = \beta\, I_i / N_i$, and a daily step applies the reaction to every
city and then the mobility coupling before advancing to the next day.

The mobility step is where the multilayer map earns its keep. Treating all travel as a single
diffusive process is a known source of error~\cite{balcan:recurrent}, so the engine
implements the two mechanisms matched to the physics of the layers, and we commit to this pair
rather than collapsing them into one:

\begin{itemize}
  \item \textbf{Diffusive mobility (air, water).} A fraction of each compartment leaves city $i$
  for neighbour $j$ each day, proportional to the layer travel rate $\tau_\ell$ and the
  normalised edge weight $w_{ij}$, and mixes into the destination population. This is the
  standard reaction--diffusion coupling and carries a global invasion threshold in mobility
  rate~\cite{colizza:invasion}.

  \item \textbf{Recurrent mobility (land, commuting).} Commuters travel to a neighbour by day,
  mix there, and return home at night, so they are not relocated. We model this by coupling the
  force of infection rather than moving population: residents of $i$ experience
  $\pi_i = \sum_j C_{ij}\, I^{*}_j / N^{*}_j$, where $C$ is the per-capita commuting matrix (the
  land kernel with a stay-home diagonal) and $I^{*}_j,\,N^{*}_j$ are the infectious and total
  people present at $j$. When no land layer is present $C$ is the identity and
  $\pi_i = I_i/N_i$, recovering the plain diffusive model.
\end{itemize}

The distinction matters: diffusive travel relocates individuals, whereas recurrent commuting
returns them home each day and so couples neighbouring cities far more tightly. The two layers we
add to air therefore play opposite roles, water spreading the outbreak diffusively to new coastal
cities while land synchronises whole regions through commuting. An optional extension lets
transmission occur during travel (most relevant for long ferry crossings) but is disabled in the
runs reported here.

\subsection{Choosing what to spread} \label{sec:spread}
The engine runs whichever compartmental model it is given, so the choice of diseases is the
substantive decision. Rather than commit to named pathogens, whose exact parameters are endlessly
contestable, we define five disease \textbf{types}, each fixed by its dynamical structure and
parameterised from one representative exemplar with values drawn entirely from the literature
(Table~\ref{tab:archetypes}). The five span the behaviours relevant to the question: an immunizing
acute epidemic (measles-like, SIR), a latent one (COVID-like, SEIR), a lethal infection controllable
by isolation (Ebola-like, SEIQRD), an endemic infection conferring no immunity (gonorrhea-like,
SIS), and a recurrent one with waning immunity (influenza-like, SEIRS). Each type adds one mechanism
relative to a simpler one, so a difference in outcome is attributable to that mechanism rather than
to several at once. All five are run and reported; where a single contrast makes a point most
sharply we lead with the two extremes, the immunizing SIR and the endemic SIS, and read the three
intermediate types against them.

\subsection{Choosing how to defend} \label{sec:defend}
With the disease types fixed, the remaining variable is the intervention, and each intervention is
a decision about which structural elements of the network to remove. We compare two modes. The
first is node targeting (vaccination): a set of cities is rendered immune so they no longer
transmit. The targeting rule is the object of study, and at a fixed budget we compare no
intervention (control), random selection, degree (the highest-traffic cities), betweenness (the
structural bridges), a motif rule we label \emph{subgraph}~\cite{estrada:subgraph,milo:motifs}, which ranks cities by
their embedding in dense local clusters (triangle participation, the number of closed triads through
each node) rather than by raw connectivity, and two optimal-percolation rules from the modern
immunization literature:
Collective Influence~\cite{morone:influence}, which scores a city by the summed reduced degrees of
the nodes on a ball of radius two around it, and a non-backtracking
rule~\cite{radicchi:nonbacktracking}, which ranks cities by their weight in the leading eigenvector
of the Hashimoto (non-backtracking) matrix. Subgraph centrality weights a node by its participation
in closed walks, so it is high precisely where a city sits in a cohesive, triangle-rich cluster in
which infection can recirculate over redundant paths rather than pass through once~\cite{estrada:subgraph}.
This is the structural signature of local transmission reinforcement: where degree counts a city's
routes and betweenness the paths it bridges, subgraph centrality identifies the tightly
interconnected neighbourhoods that sustain an outbreak once it arrives, and these need not be the
busiest or the most central cities. It therefore probes a mechanism of spread the other rules miss,
which is why we include it in the comparison. The two optimal-percolation rules are designed to
reward exactly the low-degree, high-impact cities of the anomalous-gateway kind rather than raw
hubs, and so test whether a method built for uncorrelated networks gains anything on a correlated,
real one. Having found the most effective rule, we then sweep its
budget, from a handful of cities to a few hundred, to trace the dose--response: how many cities must
be vaccinated for substantial containment, and where the returns diminish. Coverage and per-city
efficacy are held fixed throughout, so only the budget and the rule vary.

The second mode is edge targeting (interdiction): we close routes rather than immunise cities, in
four scenarios: (A) no intervention; (B) close all air routes while retaining land and water; (C)
close all air routes within an air-only model; and (D) close only the top-$k$ airports (removing all
of their routes) ranked by degree versus by betweenness. Scenarios B and C exist to contrast what a single-layer model
concludes with how the full multilayer network behaves. The two modes are distinct levers, not
stacked, so each reads in the same currency: containment per unit of a fixed budget.

\subsection{Parameters and protocol} \label{sec:params}
Each disease type fixes its clinical rates from the literature (Appendix~\ref{app:sources},
Table~\ref{tab:params-values}): recovery $\gamma$, and where present incubation $\sigma$, isolation
$\kappa$, and waning $\omega$, each the reciprocal of a measured duration. We then set the
transmission rate $\beta$ so the within-city reproduction number matches that type's literature
$R_0$. Because each city is well mixed internally, that local number is $\beta/\gamma$ for SIR, SIS,
and SEIR, and $\beta/(\gamma+\kappa)$ for the isolation type, where isolation shortens the infectious
period. For the two extremes this gives measles $\gamma=0.13$, $R_0\approx15$, hence $\beta=1.95$,
and gonorrhea a long infectious duration $\gamma\approx0.03$ with $R_0\approx1.4$, hence
$\beta\approx0.042$; the remaining three (COVID, Ebola, influenza) follow the same rule. We
deliberately apply no $\langle k^2\rangle/\langle k\rangle$ correction: that adjustment is for
networks whose nodes are individuals, whereas here nodes are well-mixed cities. Every rate and $R_0$
traces to a specific study, listed with its source in Appendix~\ref{app:sources}.

We do not calibrate to outbreak data, since the transport network carries no ground-truth epidemic
to fit, so absolute incidence is illustrative rather than predictive. Every run is seeded inside the
largest connected component: the sparser layers (ferries especially) bring in coastal cities that
sit in small disconnected pockets, where a seed cannot reach the mainland and the outbreak fizzles,
and mixing those fizzles with full epidemics would conflate two questions, whether an outbreak takes
off and how large it grows. We therefore seed where it can spread, and measure the second. A single
uniform horizon of $1460$ days (four years) gives cross-disease comparability: the acute type peaks
and burns out within months, while the endemic type needs years to approach its equilibrium. Across
any comparison the only inputs that change are the disease type, the protection strategy, and the
layer set. We report peak active infection (as a fraction of the population), final epidemic size,
and peak day; for the lethal isolation type we additionally report cumulative deaths and deaths
averted per capita, read from its fatal compartment, and for each targeting rule the geographic
concentration of the cities it selects, summarised by a per-country Gini coefficient. To quantify
seed sensitivity, every configuration is run as an ensemble of ten random seedings of the initial
infection, and each headline number is the ensemble mean with a $95\%$ confidence interval
($\text{mean}\pm1.96\,\text{s.e.}$), so every reported difference is resolved against seed variation.
We verify the engine against analytic limits, namely single-node dynamics and the final-size
correspondence between SIR and bond percolation~\cite{newman:spread}.

These pieces form a single staged protocol (Figure~\ref{fig:protocol}), walked one factor at a time
rather than as a full grid. We first run each disease with no intervention to see how it
\emph{spreads}; then \emph{defend}, comparing every targeting rule at a fixed budget per disease and
keeping each one's best, so ``which protection wins'' is read off per disease rather than assumed
from one; then trace the \emph{dose}--response by sweeping each winner's budget. The same protocol
evaluates the edge-targeting lever, \emph{interdiction}, progressively closing routes (air, then
ferries) on the flagship. The vaccination phases run up the realism ladder (air, then $+$water, then
$+$land), so each result reads against both the disease and the realism axis. In all, the staged
protocol comprises roughly $1{,}600$ independent simulations across the eight networks, each one run
of the engine at a fixed configuration and seed; walking the factors sequentially keeps this well
below the full Cartesian grid the same axes would otherwise generate.

\subsection{Implementation and reproducibility} \label{sec:impl}
The study is a single Python codebase (NumPy and NetworkX) organised as three modules, data
retrieval, network generation, and epidemic evaluation, over one shared engine
(Appendix~\ref{app:arch}). The interface is an interactive web simulator that configures an outbreak
and streams its progression as live epidemic curves and geographic maps; the same engine and
configuration are also driven programmatically to run the staged protocol over many configurations.
Either way the engine, configuration, and output are identical, so results do not depend on how a
simulation is launched, and every run is fully determined by its configuration and seed.
